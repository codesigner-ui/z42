# 语言规范编写规范

## 规范文件职责

| 文件 | 内容 |
|------|------|
| `docs/design/language-overview.md` | 语法、类型系统、所有权模型、并发模型的 **用户视角** 描述 |
| `docs/design/ir.md` | IR 指令集、类型映射、二进制格式的 **实现者视角** 描述 |

## 修改原则

- 语法变更必须同时更新代码示例块（保持可运行 / 可解析）
- 新增 IR 指令必须包含：指令名、操作数类型、语义描述、伪代码示例
- 规范中的示例代码以 ` ```z42 ` 代码块标注，IR 示例以 ` ```  ` 裸代码块标注

## z42 示例文件 (examples/)

- 每个特性至少一个示例文件
- 文件顶部注释说明演示的特性
- 示例文件应能被当前阶段的编译器正确解析（即使不能执行）
