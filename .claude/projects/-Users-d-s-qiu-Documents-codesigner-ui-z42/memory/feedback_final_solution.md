---
name: 优先采用最终方案
description: 实现时优先选择架构上正确的最终方案，临时方案需先得到 User 确认
type: feedback
---

优先采用最终的、架构上最合理的方案，不得擅自选择临时方案（hack / quick fix / 不完整实现）。

**Why:** 用户明确表达了对最终解决方案的偏好，避免积累技术债务和重复重构。

**How to apply:** 当面临方案选择时（如 A4 Bound AST 轻量版 vs 完整版），默认选择完整的最终设计。若确实存在当前无法实现最终方案的阻塞原因，必须先向 User 说明并得到确认，才能采用临时版本。
