# 前言 · 这本书是什么

这本书是 **z42 语言、编译器、虚拟机与标准库的系统化知识库**——描述系统**当前是什么**（语法、机制、架构、设计权衡），永远保持最新，面向两类读者：

- **语言使用者 / 潜在贡献者**：学习 z42 怎么用、内部怎么运作
- **维护者（含 AI）**：不必通读源码即可理解"为什么这样设计"，从而快速 review 与实现

## 它不是什么

| 不是 | 那是什么 | 在哪 |
|------|---------|------|
| 一次迭代的提案 / 任务清单 | 变更工作区（短命，做完即弃） | `docs/spec/changes/` |
| 构建 / 测试 / 发布操作手册 | 操作指南 | `docs/workflow/` |
| AI 协作行为规范 | 开发规范 | `.claude/rules/` |
| 目录级"这个目录有啥/怎么测" | 子目录 README | 各 `src/**/README.md` |

三类文档的完整职责边界、以及"知识如何从变更沉淀进本书"的治理规则，见
[`.claude/rules/doc-system.md`](../../../.claude/rules/doc-system.md)。

## 核心约定：知识从变更「上浮」进本书

每个 change 归档时，凡改变了**语言 / 编译器 / VM / stdlib 的对外行为或内部机制**，
其知识必须上浮进本书对应章节——**本书是唯一权威，变更目录随后可删（git 留痕）**。
这条约定是本书"永不过时"的保证，也是旧 `docs/design/` 能被逐步替换清理的前提。

## 迁移状态

本书正逐步吸收旧 `docs/design/`（93 篇）。每部分的概览页有一张**迁移状态表**，
记录哪些旧文档已迁入、哪些待迁。全部迁入且校对后，删除旧 `docs/design/`。
